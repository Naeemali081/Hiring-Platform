1. Project Overview
BrainCode 360 is a skill-first technical hiring platform designed for the AI era. It replaces outdated interview methods — definitions, LeetCode puzzles, memory tests — with real-world coding assessments, AI-transparent reporting, and a single defend round.
1.1 Problem Statement
The current hiring process in 2026 is fundamentally broken:
Companies hire based on memorization, not real ability
4-5 interview rounds waste time for both sides
AI bans are unrealistic — developers use AI on the job daily
Poor hires cost companies 6+ months before they realize the mistake
Existing platforms (HackerRank, Codility) still use LeetCode-style puzzles

1.2 Solution
BrainCode 360 provides:
Real-world coding problems — custom or from curated library
Role-based difficulty — Junior / Mid / Senior auto-sorting
AI-allowed environment — but every keystroke is tracked
Copy-paste vs manual code transparency report
One defend round — candidate explains their approach live
Time limits suggested by AI based on problem complexity + role

5.3 Candidate Portal Features
5.3.1 Assessment Entry
Candidate opens unique URL — no login required
Token validated server-side — if expired or used, access denied
Candidate sees: problem title, role level, time limit, instructions
Must click "Start Assessment" to begin — timer starts server-side at this point
Cannot restart once started

5.3.2 Code Editor (Monaco)
Monaco Editor — full VS Code experience in browser
Syntax highlighting for: PHP, JavaScript, Python, Java, TypeScript, Go
Auto-complete, bracket matching, error highlighting
Language selector dropdown (candidate picks their language)
Editor cannot be disabled or minimized
Full-screen mode enforced on mobile

5.3.3 Timer System
Timer displayed prominently — counts down
Timer state managed server-side (started_at + time_limit stored in DB)
Frontend syncs with server every 30 seconds via WebSocket
Warning shown at 10 minutes remaining
Warning shown at 5 minutes remaining
Auto-submit triggered when timer hits 0 — whatever is in editor is saved
Candidate can manually submit before timer expires

5.3.4 Activity Tracking
All tracking is silent — no UI indication to candidate:
Every paste event logged (timestamp + pasted content length)
Manual keystroke count tracked via input events
Tab visibility changes tracked (document.visibilityState)
Full activity log streamed to server via WebSocket in real-time
Even if candidate closes tab — partial log is already saved

5.3.5 Code Submission
Candidate clicks Submit or timer expires
Confirmation dialog shown (cannot undo)
Code saved to database + S3
If test cases exist — Judge0 executes code, result saved
Submission page shown: "Your assessment has been submitted successfully"
Assessment token marked as used — cannot be reopened

5.4 Defend Round
Company schedules defend round from dashboard
Calendar invite sent to candidate via email
Platform generates auto-suggested questions based on submitted code
Questions focus on: approach reasoning, alternative solutions, edge cases
Interviewer marks outcome: Pass / Fail / Undecided
Notes saved to submission record
Optional: live coding micro-task — interviewer can paste a modification challenge

7. User Flows
7.1 Company — Create & Send Assessment
Login → Dashboard → New Assessment
  → Select Role Level (Junior/Mid/Senior)
  → Browse Problems (filtered) OR Add Custom Problem
  → Enter Candidate Name + Email
  → Review time limit (AI suggested — can override)
  → Click "Send Assessment"
  → System: creates token, sends email, status = pending
  → Company sees assessment in list with "Pending" badge

7.2 Candidate — Complete Assessment
Receives email with unique link
  → Opens link → Sees problem + instructions + time limit
  → Clicks "Start Assessment"
  → Timer starts (server-side)
  → Monaco editor opens with selected language
  → Candidate writes code (all activity tracked silently)
  → Submits manually OR auto-submit at time=0
  → Sees confirmation page
  → Token marked as used — cannot reopen

7.3 Company — Review Submission
Notification: "Assessment submitted by [Candidate Name]"
  → Click to view report
  → See: code, execution result, transparency chart
  → Manual: X chars | Pasted: Y chars | Tab switches: Z
  → Decide: Schedule Defend Round OR Reject
  → If defend: click "Schedule" → calendar link sent to candidate
  → After defend: mark Pass/Fail + notes
  → Assessment marked as Reviewed

7.4 Edge Cases & Handling
Edge Case
System Behavior
Candidate closes browser mid-test
Activity saved via WebSocket. Timer continues server-side. Can reopen same token before expiry.
Timer expires before submission
Auto-submit triggered. Whatever is in editor is saved as final submission.
Candidate submits empty code
Allowed. Report shows 0 chars. Company sees empty submission.
Same token used twice
After first submission, token is invalidated. Second access shows "Assessment already completed" page.
Internet disconnects during test
Activity log buffered locally. On reconnect, buffered events sent to server. Timer already running server-side.
Company deletes assessment
Soft delete — data retained for 90 days for compliance.
Code execution timeout
Judge0 returns timeout error. Saved in execution_result. Report shows "Execution timed out".
